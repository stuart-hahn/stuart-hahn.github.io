How to Check if a String Is a Palindrome in Javascript

This problem and other common coding interview questions (along with their solutions) can be found in this Github Repo

Instructions: If you’re given a string, return true if the string is a palindrome and false if it isn’t.

Note: A palindrome is a word (or string) that is the same even after being reversed. Like “racecar” and “tacocat”, for example.

Reverse the string and check for equality

The most straightforward way to solve this problem is to simply reverse the string and check whether or not the reversed string is equal to the original.

This requires the knowledge of how to reverse a string but then becomes a simple equality check.

The strict equality operator === checks if two operands are equal and returns either true or false.

const palindrome = str => {
  // 1. reverse the string
  const reversedString = str
    .split('')
    .reverse()
    .join('')

  // 2. check whether the reversed string is equal to the original
  return str === reversedString
}

Without the comments (and using ES6 shortened arrow function syntax):

const palindrome = str =>
  str
    .split('')
    .reverse()
    .join('') === str

Using Array.prototype.every()

The every method checks every element against a test that you provide as a callback function and returns a boolean.

  • true if every element in the array passes the test
  • false if any element in the array fails the test
  • Calling this method on an empty array will return true for any condition.
const palindrome = str => {
  return str.split('').every((char, index) => {
    return char === str[str.length - index - 1]
  })
}

This method splits the string into an array and then calls the every method on it. The provided callback function checks each character of the array with its counterpart on the other side of the array.

The first letter is compared to the last letter. The second letter is compared to the second to last letter. Etcetera.

The big issue with this solution is that every letter gets checked twice, so the function is doing twice as much work as is really necessary.

Still, it’s good to show that you know the every method and that you understand the downsides of using an inefficient solution like this.
