Working With Class Based Components

Note: I built a little app to demonstrate how to work with class-based components. You can view the repository here: Github Repo

React has evolved through the years, and as a result functional components are much more powerful than they used to be.

In the past, functional components could only be used to produce JSX to render to the screen, and advanced functionality like lifecycle methods and state were limited to class-based components.

Modern React makes much greater use of functional components and class-based components aren’t as necessary, but it’s still important to understand what they are and how to work with them.

What is a class-based component?

A class-based component is a React component extends functionality from a React class called Component.

You can write a class based component like this:

import React from 'react'

class App extends React.Component {
	render() {
		return <div>Some JSX</div>
	}
}

export default App

They’re very similar to functional components except they have direct access to “state” and “lifecycle methods”.

“State” is a system that allows a component to keep track of certain data and change its behavior or appearance based on that data.

“Lifecycle methods” allow a component to execute code at specific times, like when it first gets mounted or just before it is unmounted.

Modern React has ways to allow functional components to do this same sort of thing, but for now let’s stay focused on class-based components.

One of the main reasons to use a class-based component is the easy management of the component’s state.

State is an object that contains data that can affect the look and behavior of a component, and class-based components make it easy to work with.

You can set the initial state simply by defining a state object, like SO:

class App extends React.Component {

	state = {
		location: "The Ozarks"
	}

	render() {
		return <h1>{this.state.location}</h1>
	}
}

Quick Note: The above code is a modern syntax for initialing state, but it’s not the only (or the original) way. For the sake of being thorough, I’ve included the outdated constructor way below.

class App extends React.Component {
	constructor(props){
		super(props)
		
		this.state = {
			location: "The Ozarks"
		}
	}
}

Things to Know About State

State is:

  • Only usable with class components (but you can get around this limitation with hooks).
  • An object that contains data relevant to a component. This object must be initialized when the component is created.
  • Changing state on a component will cause that component to re-render immediately.
  • NOT mutable. The only way to update state is using the setState() function.

LifeCycle Methods

Another feature of class components is their ability to work with lifecycle methods. Lifecycle methods let us fine-tune exactly where some code gets executed in the “life” of our component.

For example, we can use a lifecycle method called componentDidMount() to execute some code the first time our component gets “mounted” (rendered to the screen).

There are a few lifecycle methods to learn and use, but three important ones to understand are:

  1. componentDidMount - executes the first time a component is mounted in the DOM.
  2. componentDidUpdate - executes every single time a component changes (like when state gets updated)
  3. componentWillUnmount - executes right before a component gets removed from the screen. This is useful for cleanup to prevent memory leaks.

Lifecycle Methods, if defined, will automatically execute at the specific time(s) they’re supposed to.

componentDidMount() {
	// whatever is in here will automatically get executed
	// whenever this component is mounted
}

Conclusion

Class-based components aren’t as important as they used to be due to improvements to functional components, but they do still provide a simple way to work with state and lifecycle methods in React.

Modern React apps might get away with never using class components, but older codebases rely heavily on the features that only class components could provide.

I wrote a little application using only class-based components. It uses the geolocation API to grab a user’s latitude, and then uses the latitude to determine what season it is. Then, depending on the season, it will display a summer message or a winter message.
